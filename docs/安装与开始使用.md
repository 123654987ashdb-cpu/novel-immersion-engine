# 安装与开始使用

这套系统由工作流和三类技能组成：

| 部分 | 作用 |
| --- | --- |
| `workflow` | 保存一本小说的素材、规划、状态、交接包、正文、验收记录、作者改稿复盘与经验包 |
| `skill/novel-immersion-engine` | 策划端：接管项目、推演、制作交接包、验收和校准 |
| `skill/novel-prose-renderer` | 正文端：把交接包写成小说正文，避免说明书腔 |
| `skill/novel-revision-learning-loop` | 学习端：复盘作者改稿，沉淀总体创作经验与文笔执行经验 |

## 1. 安装技能

将 `skill` 下的三个技能目录安装到你的 AI 工具的技能目录：

| 工具 | 目标位置 |
| --- | --- |
| Codex | `~/.codex/skills/novel-immersion-engine`、`~/.codex/skills/novel-prose-renderer`、`~/.codex/skills/novel-revision-learning-loop` |
| Claude Code | `~/.claude/skills/novel-immersion-engine`、`~/.claude/skills/novel-prose-renderer`、`~/.claude/skills/novel-revision-learning-loop` |

技能安装成功后，常用调用名为 `$novel-immersion-engine`、`$novel-prose-renderer`、`$novel-revision-learning-loop`。

## 2. 新建一本小说

### Windows

双击 `workflow/新建小说项目.cmd`，按照提示输入项目名称和素材类型。

### macOS / Linux / Windows 命令行

```bash
python workflow/新建小说项目.py --name "我的小说" --mode "初始想法"
```

三种素材入口：

| 模式 | 适用情况 |
| --- | --- |
| `初始想法` | 只有设定、人物、画面或题材，还没有完整大纲 |
| `已有大纲` | 已有自己的故事路线，希望加强结构与落地细节 |
| `已有正文` | 已写出若干章节，希望先反向整理，再可靠续写 |

新项目默认生成在 `workflow/项目库/项目名`。公开仓库已忽略这里的新内容，因此你不容易误提交自己的小说正文。

## 3. 开始策划

将你的素材放入新项目的 `01-输入素材`。用 Codex 或 Claude Code 打开项目文件夹，发送：

```text
请使用 $novel-immersion-engine 接管本小说项目。先读取《00-项目运行规则.md》《00-项目控制台.md》和《01-输入素材》，按控制台的下一步继续工作。不要默认直接写正文；把正式策划产物写回对应目录，并更新控制台，不要只在聊天中输出结果。
```

策划端将按项目状态推进，一般会经历：

```text
小说圣经与总纲候选
  -> 你选择方向
  -> 分卷纲
  -> 当前卷阶段纲
  -> 当前阶段章节纲
  -> 阶段正文交接包
```

## 4. 生成正文

当策划端生成《阶段正文交接包》后，将交接包发给 GPT 或 Claude 的正文会话。推荐使用 `$novel-prose-renderer` 或 docs 中的正文执行提示。正文端不重新决定整书结构，而是在蓝图指定的边界内发挥语言、场景和细节创造力。

如果作者改过 AI 初稿，把交接包、AI 初稿、作者终稿交给学习端：

```text
请使用 $novel-revision-learning-loop 做作者改稿复盘。对照交接包、AI 初稿和作者终稿，提炼总体经验增量与文笔经验增量，并更新去内容化的经验包。
```

正文写回项目的 `06-待验收正文` 后，再让策划端验收：

```text
请使用 $novel-immersion-engine 验收 `06-待验收正文` 中的新正文，对照交接包和当前故事状态，判断可接受内容、需要修订的偏移以及应写回的状态增量，并更新控制台。
```

## 5. 不要跳过阶段复盘

一个阶段正文完成后，或最多两个相邻阶段完成后，应让策划端检查实际正文带来的变化，并校准尚未写出的后续规划。已经接受的正文优先于旧计划，这正是工作流既能维持长线，又不扼杀写作惊喜的关键。

更完整的路线、模板和提示词见 [完整使用说明](完整使用说明.md)。
