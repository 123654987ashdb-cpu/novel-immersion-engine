# Project Overlay Example: Emotional Cost And Witness Records

Use this reference as a pattern when a novel depends on a measurable personal erosion, transformation, contamination, memory loss, or similar accumulating cost. Adapt the fields to the actual project. It extends the general workflow; it does not replace the novel bible, ledgers, chapter contracts, scene cards, or authoritative story state.

## Activation Gate

Do not make an elegant mechanism canon merely because it would be useful. Confirm each project-specific item from written chapters or explicit author selection:

```markdown
| Hypothesis | Confirmed by source chapter/author? | Evidence | Status: candidate/canonized/rejected/revised |
| --- | --- | --- | --- |
| The deterioration has a measurable indicator |  |  |  |
| The loss mechanism is a central retention engine |  |  |  |
| A record object exists and carries emotional meaning |  |  |  |
| A witness detects changes others avoid |  |  |  |
| An investigator/explainer is at risk of exceeding their knowledge |  |  |  |
```

## Placement In The Five-Level Outline

- **小说总纲**: decide what the accumulating loss finally means for the protagonist, relationships, and ending.
- **分卷纲**: decide the irreversible threshold or relationship consequence reached in each volume.
- **卷内阶段纲**: distribute visible deterioration, witness responses, denial, and record changes across consecutive chapters.
- **章节纲**: name the loss signal, promise movement, and ending pull carried by each chapter.
- **章节写作蓝图**: specify the concrete pause, error, missing response, changed record, or witness reaction that allows the cost to be felt in prose.

## Emotional Cost Ledger

Keep only fields the project has confirmed or selected:

```markdown
| 章节 | 可测指标 | 情感/行为异常 | 本章损耗事件 | 记录物变化 | 见证者反应 | 主角是否察觉 | 对关系/选择的后果 |
| --- | --- | --- | --- | --- | --- | --- | --- |
```

Rules:

- A numeric indicator matters only when paired with a visible personal consequence.
- Escalate signs rather than repeating them: delay, mistake, altered choice, lost recognition, lost meaning.
- A record object should carry emotion as well as information: what is omitted or crossed out may matter as much as what is written.
- A witness should function as an active relationship presence or alarm, not decoration.

## Key Object Candidate

If a record object exists or is deliberately selected, place it in the Key Object Ledger:

```markdown

```

## Chapter Hook Check

At the end of relevant chapters, answer:

```markdown
- 本章损耗是否有可感知的进展、回响或预兆？
- 读者下一章想追的是“他/她还保有多少自己”，还是“下一次会失去什么”？
- 记录物写下了什么、划掉了什么，或不敢记录什么？
- 见证者察觉到了什么，而人物仍在回避什么？
- 结尾牵引是否来自真实后果，而不是空悬疑？
```

## Scene Knowledge Boundaries

For scenes involving diagnosis, concealed deterioration, investigation, or observation, add:

```markdown
- 主角知道：
- 主角不知道/不愿承认：
- 亲密关系角色知道：
- 亲密关系角色误以为：
- 见证者通过行为表现出的感知：
- 调查者有证据能知道的：
- 调查者只能推测、不得直接断言的：
- 读者比人物多知道/少知道的：
```

Do not turn an investigator into an omniscient author mouthpiece merely because they have expertise.

## Theme Delivery Rule

Do not let a character deliver the finished thematic conclusion in dialogue. Land the loss through delayed responses, missing familiarity, altered records, changed choices, relationship reactions, and the cost of apparently rational behavior.

## Reverse Outline Extraction

When continuing from existing prose, extract evidence before planning future deterioration:

```markdown
| 章节 | 实际发生事件 | 损耗证据 | 可测信息 | 记录物变化 | 见证者反应 | 调查者认知边界 | 尚未回收的承诺 |
| --- | --- | --- | --- | --- | --- | --- | --- |
```

Written evidence outranks an attractive future design.
