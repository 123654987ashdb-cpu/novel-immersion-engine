# 章节写作蓝图与正文交接

本文件用于把策划端已经完成的多层推演，收束为可直接交给 GPT 或 Claude 写正文的固定格式文件。蓝图必须来自已确认的卷内阶段骨架；长篇中的一章不得孤立设计。

## 职责分工

- **策划端**：Codex 或 Claude Code。负责故事结构、人物逻辑、伏笔回收、信息边界、节奏判断、物品、能力/物品效果与资源状态、冲突消解和正文验收。
- **正文端**：GPT 或 Claude。负责按照既定蓝图写出有表现力的正文。
- **用户**：决定方向、挑选方案、接受或否决正文与重要发现。

原则：

```markdown
需要推理和决定的事，留在策划端完成。
需要语言、场景与感受呈现的事，交给正文端发挥。
蓝图规定戏剧责任与不可越界的事实，不代替正文端提前写完每一个动作、意象和瞬间。
```

## 后台材料与交接材料

策划端内部可以维护完整材料：

- 小说圣经、总纲、分卷纲、当前故事状态。
- 人物/关系弧线、伏笔债务、认知边界、关键物品、能力/物品效果、资源、支线、节奏和声音记录。
- 变体方案、冲突处理和状态增量。

不要把整套后台材料原样塞给正文端。正文端默认接收：

1. 一份紧凑的《当前阶段导航》，使它知道本章在连续章节中的位置。
2. 一份或少量几份目标章《章节写作蓝图》。
3. 必要时附带上一章结尾的短片段，用于承接声音和现场状态。
4. 必要时附带极短的固定口吻约束，不附整套项目档案。

## 出蓝图前必须解决

在输出蓝图前，策划端先完成这些判断：

- 本章所属的小说总纲、分卷纲和卷内阶段纲路线已经选定。
- 本阶段前后连续章节的骨架已经能说明推进方向与节奏分工。
- 本章在阶段结构中的作用已经确定。
- 关键事件、必要转折与结尾牵引已经确定；只有因果脆弱或连续性高风险时，才把精确场景路径锁死。
- 人物为何这样选择，以及选择后的状态变化已经确定。
- 本章要推进/回收/压住的伏笔已经确定。
- 信息边界、关键物品、能力/物品效果、资源或核心代价变化已经确定。
- 最近章节重复风险已处理。
- 与 canon、作者长期意图或当前焦点的冲突已解决，或明确交给用户决定。

未解决的关键问题不得伪装成蓝图里的既定剧情。

## 受控想象力：蓝图不是填空题

策划端要保护结构，也要保护正文的生命力。每份蓝图按照三层自由度交付：

| 层级 | 含义 | 例子 |
| --- | --- | --- |
| 固定锚点 | 不改变，否则故事或人物逻辑会坏掉 | 必须发生的选择、揭示、关系结果、关键物品/能力边界/代价状态、结尾落点 |
| 边界内创造 | 结果与边界已定，正文端自己寻找最好的抵达路径 | 角色怎样周旋、争执如何升级、场面里出现什么即时阻碍、如何把情绪藏进动作 |
| 开放创造位 | 只给必须产生的感觉或功能，不预写实现方法 | 一处让读者心软的细节、一处反常的不安、一处关系变近却无人承认的瞬间 |

规则：

- 每章在不损害逻辑时，至少留一个开放创造位；因谜底、精确诡计或强连续性必须严控的章节可以说明例外。
- 不把本来可以由正文端创造的动作、对白、意象和微型冲突全写成指定答案。
- 正文端可以在边界内写出比策划端预想更好的局部发现；只要它不新增必须由后文偿还的重大承诺，不改关键状态，就不是越界。
- 正文验收时，把有效发现视为可保留的作品财富；若发现自然牵动后续路线，再由策划端与用户决定是否纳入规划。

## 阶段正文交接包格式

正文交接默认先给阶段导航，再给目标章蓝图。若只生成一章，也保留简短阶段导航。

```markdown

- 本阶段覆盖章节：
- 阶段开始状态：
- 阶段结束时必须达到的状态：
- 阶段核心矛盾与主要回报：
- 本阶段人物/感情变化方向：
- 本阶段重要伏笔安排：

## B. 当前需生成正文的章节
- 写作范围：第 X 章 / 第 X-Y 章
- 要求：按下方每章蓝图生成正文，不得改写阶段走向。

## C. 第 X 章《章节名》章节写作蓝图

- 输出本章完整正文，不附分析、总结或修改说明。
- 严格遵守固定锚点、场景因果与既定结尾；在边界内创造与开放创造位中主动写出具体、自然、意外而贴合人物的执行。
```

若用户要一次写同阶段的多章，在同一《当前阶段导航》后连续附多份《章节写作蓝图》。验收和状态结算仍逐章进行。

## 蓝图质量检查

交给正文端前，确认：

- 正文端不需要自行决定关键剧情。
- 正文端能从阶段导航看懂当前章前后为何如此安排。
- 当前章没有破坏分卷纲和阶段纲已经确定的走向。
- 每个场景都写清了“发生什么、为什么发生、如何变化”。
- 必须呈现的效果可通过动作、对白、感官或物品具体落地。
- 固定锚点与禁止越界事项足够明确，但未把可创造的场景路径提前写死。
- 蓝图提供了边界内创造和开放创造位，正文端不会只剩填空工作。
- 蓝图只含本章必要材料，没有倾倒后台账本。

## 正文回流验收

正文端完成后，策划端负责验收：

1. 是否执行了本章既定事件与场景转折。
2. 是否实现预定回报、人物变化、关系变化与结尾牵引。
3. 是否擅自新增设定、泄露信息、改变物品/资源、越界使用或临时升级技能/物品效果，或让角色越界。
4. 是否在创造空间内写出了值得保留的局部发现、意象、动作或关系质感。
5. 通过后只结算确认发生的状态增量，再制作下一章蓝图。
