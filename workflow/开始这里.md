# 小说沉入工程工作流

> 整套系统的统一入口已整理至：[小说沉入工程/开始使用.md](<D:/写作skill/小说沉入工程/开始使用.md>)。本文件专门说明工作流本身。

这是一套让小说长期运转起来的本地工作台，已将正式技能的主要能力映射为项目文件。**工作流是小说项目本体，`小说沉入工程` 技能是驾驶它的策划引擎。** 技能负责推理、制作产物、验收与推进；项目负责保存事实、选择、正文、作者改稿学习与下一步。即使交给没有安装技能的 AI，新项目内的《项目运行规则》也会说明应怎样工作。

## 最简单的开始方式

1. 双击 `新建小说项目.cmd`。
2. 输入小说名，并选择你当前有的是“初始想法”“已有大纲”还是“已有正文”。
3. 新项目会出现在 `项目库` 文件夹中。
4. 把想法、大纲或正文放进项目的 `01-输入素材`。
5. 把整个项目文件夹交给 Codex 或 Claude Code，并说：

```text
请使用 $novel-immersion-engine 接管本小说项目。先读取《00-项目运行规则.md》《00-项目控制台.md》和《01-输入素材》，按控制台的下一步继续工作。不要默认直接写正文；把正式策划产物写回对应目录，并更新控制台，不要只在聊天中输出结果。
```

## 接管关系

| 角色 | 负责什么 |
| --- | --- |
| 小说项目文件夹 | 保存控制台、正式规划、canon、账本、交接包、验收与复盘 |
| Codex / Claude Code 加载的策划技能 | 读取项目、完成推理、写回产物、推动下一节点 |
| GPT / Claude 正文会话 + 正文技能 | 按交接包和文笔经验包写出正文，不重做结构决定 |
| 改稿学习技能 | 对照蓝图、AI 初稿、作者终稿，提炼总体经验与文笔经验 |
| 你 | 选择方向、判断审美、决定重大改线与发现是否纳入故事 |

如果你准备长期写一本书，就从项目文件夹开始；如果只是讨论一个想法，可以暂时不建项目。

## 整体运行方式

```text
你的素材
  -> 策划端建立小说圣经和总纲候选
  -> 你选择方向
  -> 策划端建立分卷纲与当前卷阶段纲
  -> 策划端制作内部执行包/场景卡与阶段正文交接包
  -> GPT / Claude 写正文
  -> 你修改正文
  -> 学习端复盘作者改稿并沉淀经验
  -> 正文放回待验收区
  -> 策划端验收并更新当前事实
  -> 一个阶段完成后复盘与校准
  -> 一卷完成后卷末复核并继承到下一卷
  -> 下一阶段继续
```

## 文件夹分别负责什么

| 位置 | 用途 |
| --- | --- |
| `项目库` | 每一本正在创作的小说 |
| `模板` | 新建项目时自动复制的空白项目结构 |
| `提示词` | 可直接复制给策划端与正文端的常用话术 |
| `工具` | 新建项目的内部工具 |

## 三条最重要的原则

1. 已经验收接受的正文，优先于旧计划；后续计划可以被校准。
2. 交接包锁住关键锚点，但必须留出边界内创造和开放创造位，让正文端能写出生命力。
3. 正式项目的成果必须写入项目文件，不能只散落在与 AI 的聊天里。

## 已经内置的控制材料

- `04-当前故事状态/账本`：按需启用的人物、关系、伏笔、信息、代价、物品、技能/物品效果、资源、节奏与冲突记录。
- `05-正文交接包/完整交接包模板.md`：整章或连续章节正文的正式出口。
- `08-验收与状态增量`：正文回来后的逐章判定与事实写回。
- `09-阶段复盘与校准`：阶段完成后的路线校准与卷末继承。
- `11-作者改稿学习与经验包`：保存作者改稿复盘、总体经验增量、文笔经验增量、整体创作经验包和文笔执行经验包。
- `完整映射清单.md`：查看技能能力在项目中分别落在哪里。
