# 工作流蓝图

## 定位

本工作流是小说项目的持续运行本体；`小说沉入工程` 技能是使用它、驾驶它的策划引擎。两者不是重复关系，而是“项目容器 + 推理执行器”的关系：

- 作者作方向选择与审美判断。
- Codex / Claude Code 加载技能后作为策划端，接管项目状态、生成并写回规划、制作交接包、验收正文和校准路线。
- GPT / Claude 加载正文技能或正文提示后作为正文端，读取交接包与文笔执行经验包并写正文。
- 改稿学习端读取蓝图、AI 初稿和作者终稿，提炼总体经验与文笔经验。
- 项目文件夹保存已经发生的事实、选中的路线、正文版本、改稿复盘和经验包。

## 技能接管协议

1. 找到小说项目中的 `00-项目运行规则.md` 与 `00-项目控制台.md`。
2. 技能先读取两者，根据控制台当前节点加载必要文件。
3. 正式规划、交接、验收和复盘产物写回指定目录，聊天只汇报结果和需要作者决定的事。
4. 没有项目而用户准备长期创作时，先建立项目再开始正式规划。
5. 单次讨论或临时试验可以不建项目，但不能冒充已确认的项目状态。

## 状态流转

| 状态 | 当前任务 | 产物存放处 | 进入下一步的条件 |
| --- | --- | --- | --- |
| S0 项目接入 | 接收想法、大纲或已有正文 | `01-输入素材` | 策划端确认接入路线 |
| S1 总纲选择 | 小说圣经与三个总纲方向 | `02-小说圣经与总纲` | 作者选择或合并方向 |
| S2 分卷规划 | 已选总纲与分卷纲 | `02-小说圣经与总纲`、`03-分卷与阶段规划` | 当前卷方向成立 |
| S3 阶段规划 | 当前卷 3-4 个阶段与当前阶段章节纲 | `03-分卷与阶段规划` | 作者接受当前阶段 |
| S4 正文交接 | 当前章或小批章节交接包 | `05-正文交接包` | 发给正文端生成 |
| S5 正文验收 | 检查返回正文与状态变化 | `06-待验收正文`、`08-验收与状态增量` | 章节接受或退修 |
| S6 阶段复盘 | 阶段结果与后续校准 | `09-阶段复盘与校准` | 下一阶段路线确认 |
| S7 卷末复核 | 本卷兑现与下一卷继承 | `03-分卷与阶段规划`、`04-当前故事状态` | 开始下一卷或完结 |
| S8 作者改稿学习 | 从作者修改中提炼经验 | `11-作者改稿学习与经验包/01-作者改稿复盘`、`02-经验增量` | 经验增量写入 |
| S9 经验打包 | 将多次经验去内容化为可迁移方法 | `11-作者改稿学习与经验包/03-经验包` | 更新整体与文笔经验包 |

## 策划端每次接手项目时

1. 先读取 `00-项目运行规则.md` 与 `00-项目控制台.md`。
2. 只加载当前动作需要的资料，例如总纲选择不必读取所有正文，正文验收则必须读取蓝图、返回正文和当前状态。
3. 把新产物写入约定目录，不用聊天输出代替正式文件，也不把候选内容混进已确定事实。
4. 只有作者选择或正文验收通过后，才更新 `04-当前故事状态/当前故事状态.md`。
5. 制作新交接包前读取适用的 `整体创作经验包`；正文端写新正文前读取适用的 `文笔执行经验包`。
6. 每完成一步，更新控制台中的状态、最新产物、待决定问题和下一动作。

## 交接包的创造自由

交给正文端的材料不应成为填空题：

| 层级 | 策划端处理方式 |
| --- | --- |
| 固定锚点 | 明确列出，正文不可改变结果 |
| 边界内创造 | 指明边界和目标，让正文端自行设计抵达过程 |
| 开放创造位 | 只说明需产生的感受或功能，主动等待正文端的惊喜 |

## 正文回流规则

- 正文端输出先进入 `06-待验收正文`，不是 canon。
- 策划端逐章验收，通过后移入 `07-已接受正文` 并记录状态增量。
- 一个阶段完成后默认复盘；连续写了两个阶段时最迟必须复盘。
- 已接受正文不能被旧计划覆盖；只修改尚未生成的后续阶段。
- 作者终稿可以触发学习闭环；学习结论不等于 canon，只有经验包会反哺下一轮策划和正文执行。

## 第一版边界

第一版专注把项目资料、状态和模型协作跑通，不自动调用 GPT 或 Claude API，也不做复杂界面。这样能先验证小说流程是否真的提升正文质量，避免在未经验证前把时间耗在软件外壳上。

## 节点通过条件

| 节点 | 不可跳过的通过条件 |
| --- | --- |
| S0 项目接入 | 知道素材类型、作者不可丢失的核心和已处理/未处理范围 |
| S1 总纲选择 | 小说圣经成立；候选方案可比较；作者选择或明确合并方案 |
| S2 分卷规划 | 已选总纲能闭环；每卷有独立变化、回报和向后递交的问题 |
| S3 阶段规划 | 当前卷拆成连续阶段；目标阶段有起止状态、因果链、章节职责和节奏检查 |
| S4 正文交接 | 当前章有合同和必要场景控制；交接包包含固定锚点、创造空间、边界与结尾落点 |
| S5 正文验收 | 正文逐章对照交接包；通过后才更新 canon 和账本 |
| S6 阶段复盘 | 当前阶段章节已验收；后续未写路线完成保持、校准或改线决定 |
| S7 卷末复核 | 本卷承诺、角色变化、能力/物品效果、资源/代价与下卷继承已结算 |
| S8 作者改稿学习 | 蓝图、AI 初稿、作者终稿已对齐；总体经验增量和文笔经验增量已拆分 |
| S9 经验打包 | 多次经验增量完成去内容化，形成可给策划端和正文端复用的经验包 |

## 按需启用而非全部填满

完全映射不等于每章都填写所有表。项目初建时由策划端在 `04-当前故事状态/账本启用清单.md` 中选择活跃模块：

- 人物弧线与关系弧线通常启用。
- 有谜团、欺骗或权谋时启用信息边界。
- 有核心损耗、倒计时或污染机制时启用核心代价。
- 物品承载证据、感情或回收时启用关键物品。
- 有功法、技能、异能、装备效果、升级突破或战斗解法时启用能力与物品效果。
- 资源稀缺或支线很多时启用资源与支线。
- 连载拉长或口吻漂移风险高时启用节奏与声音。

不用的模块保持未启用，不给创作增加无意义表格负担。
