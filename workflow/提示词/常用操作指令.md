# 常用操作指令

以下话术供你直接复制。项目运行时，把“本项目文件夹”替换成你的项目路径，或者在 Codex / Claude Code 已经打开该项目时直接发送。所有策划任务都由 `$novel-immersion-engine` 接管项目，并先读取 `00-项目运行规则.md` 和 `00-项目控制台.md`。

## 1. 接管项目并开始或继续工作

```text
请使用 $novel-immersion-engine 接管本小说项目，先读取《00-项目运行规则.md》《00-项目控制台.md》与当前节点所需材料；若目前是首次接入，再读取《01-输入素材》。
依据控制台推进，不默认写正文。请把正式产物保存到本项目对应目录，并更新控制台中的当前节点、最新产物、待我决定的问题和下一步动作。不要只在聊天中输出正式结果。
```

## 2. 我选择总纲方案以后

```text
本项目已经由 $novel-immersion-engine 接管。我选择总纲方案：[写方案编号与需要混合/删除的内容]。
请形成正式《已选总纲》，完成闭环检查，然后展开《分卷纲》。将候选与正式选择区分开，并更新项目控制台。
```

## 3. 进入当前卷与阶段规划

```text
请在已接管项目中为第 X 卷生成 3-4 个连续阶段纲，说明每个阶段的开始状态、结束状态、回报、人物/感情推进、伏笔任务和章节预估。将产物写回规划目录并更新控制台，当前先不要写正文。
```

## 4. 制作正文交接包

```text
请使用 $novel-immersion-engine 根据当前项目状态，为第 X 卷阶段 Y 的第 A 章（或第 A-B 章）生成《阶段正文交接包》，保存到 `05-正文交接包`。
先在策划端完成涉及本章的内部执行包、场景卡与启用账本核对。整章、续写或有重要状态变化时使用《完整交接包模板》，只有低风险短场景才可使用快速模板。
请明确固定锚点、边界内创造和开放创造位；如涉及技能、功法、异能或装备效果，必须写清允许效果、限制、代价、是否升级和不得临时发明的能力。不能把正文端变成机械填空，也不能让它擅自决定长线剧情。
```

## 5. 交给 GPT / Claude 写正文

```text
请使用 $novel-prose-renderer。你现在负责小说正文表现，不负责重新策划剧情。

请依据我提供的《阶段正文交接包》生成正文；若我同时提供《文笔执行经验包》，请把它作为文笔执行经验，而不是故事 canon。固定锚点、信息边界、关键状态变化和结尾落点必须遵守；对“边界内创造”和“开放创造位”，请主动寻找自然、有感染力、意外而合理的执行方式。

不得新增需要后续偿还的重要设定或伏笔，不得临时新增、升级或越界使用技能/物品效果，不得改变人物关键决定、关系转折、物品/资源/核心代价状态或隐藏信息。符合人物与世界且不制造后续负债的局部创作发现是允许的。

请只输出正文。如交接包内部存在无法兼容的矛盾，只指出矛盾。

以下是交接包：
```

## 5A. 作者改完正文后做学习复盘

```text
请使用 $novel-revision-learning-loop 对本章进行作者改稿复盘。请读取：
1. 对应的 `05-正文交接包`；
2. AI 生成的初稿；
3. 作者修改后的终稿或 `07-已接受正文`；
4. 如已有，读取 `11-作者改稿学习与经验包/03-经验包` 中的两份经验包。

请把结果写入 `11-作者改稿学习与经验包/01-作者改稿复盘`，并分别更新 `02-经验增量/总体经验增量.md` 与 `02-经验增量/文笔经验增量.md`。
注意：原始复盘可以引用本章内容；经验增量要尽量抽象；经验包更新前必须去内容化，不要泄露本书私有剧情、角色名或原文。
```

## 5B. 多次复盘后打包经验

```text
请使用 $novel-revision-learning-loop 汇总 `11-作者改稿学习与经验包/02-经验增量` 中尚未进入经验包的条目。
请将可迁移的策划方法写入 `03-经验包/整体创作经验包.md`，将可迁移的文笔方法写入 `03-经验包/文笔执行经验包.md`。
要求去内容化：不保留本书角色名、专属设定、关键剧情或原文；每条经验都要写明适用范围、置信度、使用方式和反例/边界。
```

## 6. 验收返回正文

```text
请使用 $novel-immersion-engine 验收放在 `06-待验收正文` 的最新正文，并读取它对应的 `05-正文交接包`、内部执行包、`04-当前故事状态/当前故事状态.md` 以及本章涉及的启用账本。
请输出验收记录到 `08-验收与状态增量`。如通过，请更新当前故事状态、对应账本和控制台；如需修改，请清楚列出问题和修改方向，不要擅自把未通过内容写入 canon。
```

## 7. 阶段完成后复盘

```text
当前阶段正文已完成并逐章验收。请使用 $novel-immersion-engine 进行《已完成阶段复盘与后续校准》，将结果保存到 `09-阶段复盘与校准`。
请保留已经接受的正文事实，只调整尚未写出的后续计划。若改变本卷高潮或全书方向，先给我方案选择，再继续制作下一阶段交接包。
```

## 8. 一卷完成后复核

```text
第 X 卷全部阶段已经完成验收和阶段复盘。请使用 `09-阶段复盘与校准/卷末复核模板.md` 完成本卷结算，确认本卷承诺兑现、仍欠下的债务、人物和关系继承、核心代价、关键物品及能力/物品效果状态，并判断下卷路线是否需要重新给方案供我选择。
```

## 9. 做版本对比

```text
请基于相同的小说状态和同一章任务，分别使用“V1 阶段复盘版”和“当前受控想象力版”制作两份交接包，用同一个正文模型生成正文，然后在 `10-版本对比实验` 中比较：剧情稳定性、人物自然度、惊喜感、机械感、越界风险和保留价值。
```
